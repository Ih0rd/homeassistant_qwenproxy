# Qwen OAuth Proxy — Home Assistant Addon

OpenAI-compatible прокси для подключения Qwen (portal.qwen.ai) к Home Assistant через OAuth токен. Позволяет использовать мощные модели Qwen бесплатно в качестве текстового ассистента умного дома. Аддон работает как локальный OpenAI-совместимый сервер и проксирует запросы на `portal.qwen.ai`, используя ваш OAuth токен.

## Шаг 1 — Получение OAuth credentials через qwen-code

OAuth токен генерируется через официальный CLI инструмент [QwenLM/qwen-code](https://github.com/QwenLM/qwen-code). Токен действителен 12 месяцев.

### Установка qwen-code

Требуется Node.js 18+.

```bash
npm install -g @qwen-code/qwen-code
```

### Авторизация

```bash
qwen auth login
```

Откроется браузер с OAuth страницей portal.qwen.ai. После авторизации credentials автоматически сохранятся в файл:

- **macOS / Linux:** `~/.qwen/oauth_creds.json`
- **Windows:** `%USERPROFILE%\.qwen\oauth_creds.json`

Сохраните значения `access_token`, `refresh_token` и `expiry_date` — они понадобятся при настройке аддона.

> **Важно:** `expiry_date` — это Unix timestamp в миллисекундах. Когда токен истечёт, повторите `qwen auth login` и обновите credentials в настройках аддона.


## Шаг 2 — Установка аддона в Home Assistant

### 2.1 Добавить пользовательский репозиторий

В Home Assistant:

**Настройки → Дополнения → Магазин дополнений → ⋮ (три точки) → Пользовательские репозитории**

Добавить URL этого репозитория, тип — **Add-on**.

### 2.2 Установить аддон

После обновления магазина найти **Qwen OAuth Proxy** и нажать **Установить**.

### 2.3 Настроить аддон

Перейти на вкладку **Конфигурация** аддона и заполнить:

```yaml
access_token: "ВАШ_ACCESS_TOKEN"
refresh_token: "ВАШ_REFRESH_TOKEN"
expiry_date: "ВАШЕ ЗНАЧЕНИЕ"
default_model: "qwen3-coder-plus"
```

### 2.4 Запуск аддона

Нажать **Запустить**. В логах должно появиться:

```
[entrypoint] Credentials written to /root/.qwen/oauth_creds.json
[entrypoint] Starting qwen-code-oai-proxy on port 8080...
Qwen OpenAI Proxy listening on http://0.0.0.0:8080
Default account: ✅ Valid
```

## Шаг 3 — Установка Extended OpenAI Conversation

Стандартная интеграция OpenAI в HA не поддерживает кастомный Base URL. Используйте [jekalmin/extended_openai_conversation](https://github.com/jekalmin/extended_openai_conversation) через HACS.

**HACS → Интеграции** → поиск `Extended OpenAI Conversation` → **Установить** → перезапустить Home Assistant.


## Шаг 4 — Настройка интеграции

**Настройки → Устройства и службы → Добавить интеграцию** → `Extended OpenAI Conversation`

| Поле | Значение |
|---|---|
| **Name** | `Qwen` |
| **API Key** | `qwen` (любая непустая строка) |
| **Base URL** | `http://homeassistant.local:8080/v1` |
| **Model** | `qwen3-coder-plus` |
| **Skip Authentication** | ☑️ включить |

Обязательно укажите в настройках интеграции (**Настройки → Устройства и службы → Extended OpenAI (Qwen) → Настроить**) chat_model идентичную той, что установлена в настройках Qwen OAuth Proxy.

## Рекомендуемые настройки

| Параметр | Рекомендуется | Описание |
|---|---|---|
| **Maximum tokens to return** | `8000` | Для развёрнутого анализа датчиков и создания автоматизаций |
| **Maximum function calls** | `24` | Цепочки действий: опросить датчики → принять решение → выполнить |
| **Context Threshold** | `100000` | Модель поддерживает 1M токенов входного контекста |
| **Temperature** | `0.3` | Точность важнее креативности при управлении умным домом |
| **Top P** | `0.8` | Предсказуемые и стабильные решения |
| **Use Tools** | ☑️ включить | Необходимо для вызова сервисов HA |

### Доступные модели

| Модель | Контекст | Особенности |
|---|---|---|
| `qwen3-coder-plus` | 1M токенов | Основная, лучший баланс качества и скорости |
| `qwen3-coder-flash` | 1M токенов | Быстрее, подходит для простых команд |
| `vision-model` | 1M токенов | Модель распознавания изображений |

## Открытие устройств для LLM

По умолчанию большинство устройств скрыты от LLM. Чтобы модель могла ими управлять или читать их состояние:

**Настройки → Голосовые ассистенты → Открытые объекты**

Найдите нужные устройства через поиск и включите переключатель **Expose / Открыть** для каждого.

Проверьте что у датчиков корректно задан `device_class` (например `occupancy`, `motion`, `temperature`) — без этого HA может не предлагать их для экспозиции.

## Сценарии использования

### Анализ датчиков

```
Покажи мне историю температуры в спальне за последние 3 дня
и объясни, почему она падает каждую ночь после 2 часов.
```

### Автоматизации через диалог

```
Создай автоматизацию: если датчик движения в коридоре срабатывает
после 23:00, включи ночник на 30% яркости вместо основного света.
```

> **Примечание:** Qwen может сгенерировать YAML для автоматизации, который нужно вставить вручную в `configuration.yaml` или через редактор автоматизаций HA. Прямое создание через API недоступно стандартными сервисами.

### Условные сценарии

```
Если все в доме ушли (presence_sensor = false), температура на улице
ниже 5 градусов, и время между 22:00 и 08:00 — установи
термостат на 18 градусов и выключи весь свет.
```

### Уведомления с контекстом

```
Объясни мне аномалии в потреблении электричества за сегодня
и предложи что можно оптимизировать.
```

## Обновление токена

1. На любом компьютере с установленным `qwen-code` выполнить:
   ```bash
   qwen auth login
   ```
2. Взять новые значения из `~/.qwen/oauth_creds.json`
3. Обновить в **Конфигурации аддона** → **Сохранить** → **Перезапустить аддон**
